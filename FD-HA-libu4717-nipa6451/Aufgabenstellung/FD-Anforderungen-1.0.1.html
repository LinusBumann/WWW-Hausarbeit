Frontend-Design 2022

Anforderungen – Version 1.0.1

Dieses Dokument listet die Anforderungen an die Frontend-Design Hausarbeit auf – soweit sie eindeutig zu formulieren sind.

Schlüsselworte

Die Schlüsselwörter »MUSS«, »DARF NICHT«, »ERFORDERLICH«, »SOLL«, »VERBOTEN«, »NÖTIG«, »NICHT NÖTIG«, »SOLL NICHT«, »EMPFOHLEN«, »DARF«, »KANN« und »OPTIONAL« werden nach der Interpretation des RFC 2119 in deutscher Sprache verwendet.

Allgemein

  • Ihre Namen, Login-IDs und Matrikelnummern MÜSSEN in der Dokumentation zu finden sein.

Design / Inhalte

  • Die Styletiles MÜSSEN deutlich unterschiedliche Designansätze zeigen.
  • Es MUSS eine Auseinandersetzung mit dem Design der Site deutlich werden.
  • Das Design MUSS eine mindestens durchschnittliche Gestaltungshöhe aufweisen.
  • Typografie, Farben, Bildsprache MÜSSEN das Ziel der Site unterstützen.
  • Die Inhalte MÜSSEN passend und realistisch sein.
  • Blindtexte sind VERBOTEN.
  • Singlepage-Sites sind VERBOTEN.
  • Es MUSS eine Seiten mit längerem Text vorhanden sein.
  • Es MÜSSEN unterschiedliche Seitentypen vorhanden sein.
  • Es SOLLTE eine Übersichtsseite (Ausschnitte aus mehreren Inhalten) vorhanden sein.
  • Es SOLLTEN Detailseiten (vollständiger Inhalt zu einem Thema) vorhanden sein.
  • Es SOLLTEN Variationen (z.B. mehr/weniger Inhalt/Bilder) einzelner Seiten vorhanden sein.
  • Es MUSS ein funktionierendes Formular vorhanden sein.
  • Es MUSS ein Impressum vorhanden sein.
  • Es MUSS eine Datenschutzerklärung vorhanden sein.
  • Die Rechte für Inhalte, Bilder etc MÜSSEN vorliegen und im Kollophon nachgewiesen werden.

Dateien

  • Alle Datei- und Ordnernamen DÜRFEN nur die Zeichen a-z, 1-9, ., _, - enthalten.
  • Alle Dateien MÜSSEN UTF-8 (ohne BOM) codiert sein.
  • Alle Referenzierungen auf Dateien im Projekt (z.B. Bilder, Web-Fonts, HTML-, CSS-Dateien) MÜSSEN relative Pfade nutzen.

HTML

  • Die Website MUSS mit aktuellem HTML umgesetzt werden.
  • Die Website MUSS als statische HTML-Dateien vorliegen.
  • Die einzelnen Seiten MÜSSEN miteinander verlinkt sein.
  • Der Doctype MUSS html sein.
  • In allen HTML-Dateien MUSS das Dokumenten-Encoding als Metatag angegeben werden.
  • Alle HTML-Dateien müssen ein für Responsive Webdesign passendes passendes Metatag viewport haben.
  • Das HTML MUSS valide sein. (Prüfung mit https://validator.w3.org)
  • Das HTML MUSS den Anforderungen der Web Content Accessibility Guidelines (WCAG) 2.0 entsprechen. (Soweit in der Vorlesung erläutert. Prüfung mit https://pa11y.org oder https://wave.webaim.org/extension/)
  • Die Auszeichnung mit HTML MUSS semantischen sein. (Prüfung ohne CSS und am Quellcode.)
  • Struktur, Präsentation und Verhalten MÜSSEN klar getrennt sein.
  • Alle HTML-Seiten SOLLTEN einen ›Springe zum Inhalt‹-Link aufweisen.
  • Die Daten des Formulars MÜSSEN per POST an https://www.inf.fh-flensburg.de/echo gesendet werden.
  • Es ist VERBOTEN Tabellen für Layoutzwecke zu nutzen.
  • Alle Inhalte MÜSSEN mit allen User-Agents lesbar sein.
  • Alle Inhalte und Grundfunktionen MÜSSEN ohne Javascript lesbar/nutzbar sein.
  • Javascript KANN als Progressive Enhancement für zusätzliche Interaktionen, Effekte etc genutzt werden.
  • Es ist VERBOTEN inline Javascript zu nutzen (z.B. onclick="call_js("No!");").

CSS

  • Das Design MUSS mit CSS umgesetzt werden.
  • Es ist VERBOTEN Style-Attribute zu nutzen.
  • Das CSS MUSS valide sein. (Prüfung mit https://jigsaw.w3.org/css-validator/)
  • CSS-Klassen SOLLTEN nach SUIT oder BEM benannt sein.
  • Das CSS SOLLTE nach IT-CSS organisiert sein.
  • Es ist VERBOTEN fremdes CSS oder CSS-Frameworks (wie z.B. Bootstrap, Tailwind) zu nutzen.
  • Kurze Abschnitte fremden CSS-Codes KÖNNEN genutzt werden.
  • Kurze Abschnitte fremden CSS-Codes MÜSSEN im CSS-Code gekennzeichnet sein und die Quelle MUSS im CSS-Code angegeben werden.
  • Das Design bzw. Layout MUSS sich an verschiedene Endgeräte/Bildschirmgröße anpassen.
  • Das Design bzw. Layout DARF je nach Endgerät/Bildschirmgröße unterschiedlich sein.
  • Es MÜSSEN Web-Schriften – in Form von freien Webfonts – genutzt werden.
  • Die Web-Schriften MÜSSEN lokal im Projekt vorhanden sein – d.h. keine Einbindung von externen Server (z.B. Google-Fonts) per <link src="http://externer.server> oder Javascript.
  • Es SOLLTEN Bilder genutzt werden.
  • Bilder SOLLTEN auf sinnvolle Maße beschnitten werden.
  • Für Bilder SOLLTEN passende Dateiformate genutzt werden.
  • Es DÜRFEN weitere Medien wie Audio, Video, SVG etc genutzt werden.
  • Mediaqueries SOLLTEN ›mobile first‹ (also mit min-width) sein.
  • CSS MUSS für alle Bildschirmgrößen die Gestaltung steuern.
  • Es MUSS ein simples Grundlayout ohne Grid als Fallback vorhanden sein.
  • Es SOLLTE Grid für das Seiten-Layout genutzt werden.
  • Alle Seiten der Site MÜSSEN die selbe CSS-Datei nutzen.
  • Es DARF eine weitere CSS-Datei für media='print' genutzt werden.
  • Für die Dokumentation DARF eine weitere CSS-Datei genutzt werden.
  • Für das Kontrastraster (Farben) SOLLTE eine eigene CSS-Datei genutzt werden.
  • Es DARF ein CSS-Preprozessor genutzt werden. In dem Fall MÜSSEN Quellcode und kompiliertes CSS abgeben werden. werden.

Abgrenzung

  • Severseitige Programmierung DARF NICHT genutzt werden: In HTML und CSS dargestellte Webshops, Foren etc MÜSSEN ohne Funktion sein. (Das folgt in WWW-Programmierung.)

Dokumentation

  • Die Dokumentation MUSS die Design und die Entwicklung klar und nachvollziehbar beschreiben.
  • Die Dokumentation MUSS formal (Zeichensetzung, Rechtschreibung, Struktur, Typografie) muss mindestens ausreichend sein.
  • Es SOLLTE eine Zeitleiste mit Screenshots der Designentwicklung und Umsetzung geben.
  • Es SOLLTE in der Dokumentation ein Seite mit allen HTML-Elementen/Module geben.
  • Es SOLLTE in der Dokumentation eine Seite mit den Farbkontrasten geben.

Werden die Anforderungen nicht erfüllt, wird die Arbeit deutlich abgewertet oder mit 5,0 bewertet.
